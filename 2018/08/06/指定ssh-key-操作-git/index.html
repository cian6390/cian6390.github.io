指定ssh-key 操作 git

  • Cian
  • 2 Minutes
  • August 6, 2018

最近由於比較常使用不同 Repository以及常更換設備 clone專案
因此遇到ssh-key不對的問題，情況大概就是系統預設的ssh-key跟git server上設定的不同
因此在操作的時候需要明確指定ssh-key

在網路上找的了兩個指令可以做到一樣的事情

但這樣的方式畢竟不是最好的
如果可以讓系統針對某些host 調用指定的key那就更好了

有的，那就是ssh的config

我們可以在家目錄底下的.ssh資料夾建立一個無副檔名的檔案config，如: ~/.ssh/config

接著就可以指定Host的方式來套用設定

其中git@github1的 github1就是我們在config檔案裡的 Host

透過變更 Host的方法，就可以做到對相同git server的不同Repository使用不同的 ssh-key
